# CutFlow•CAD — ロードマップ

段階リリース計画。早期に価値を出しつつ、リスクを抑えて積み上げます。

---

## Phase 0 — 基盤構築

**目的**: 開発できる状態を整える

- [ ] リポジトリ初期化、設計仕様書コミット ← **完了**
- [ ] `docker-compose.yml` (FastAPI + Vite dev)
- [ ] Vue 3 + Vite プロジェクト雛形
- [ ] FastAPI プロジェクト雛形 + ezdxf インストール
- [ ] v3 デザインのCSSトークン / SVGアイコンを `web/src/assets/` へ移植
- [ ] レイアウト骨格（ヘッダ + ツールレール + キャンバス枠 + インスペクタ枠 + タブ + ステータス）

**完了条件**: `docker-compose up` で v3 と同じ見た目の空アプリが表示される

---

## Phase 1 — DXF取込・表示・削除機能 (MVP)

**目的**: 最も即効性の高い価値を最初に届ける

- [ ] DXFファイル(複数) アップロードAPI
- [ ] サーバー側 ezdxf 読込 → エンティティJSON返却
- [ ] フロント側 SVGレンダリング（外径・穴・寸法・タップ・バルーン・図枠を色分け）
- [ ] 下部DXFタブで複数ファイル切替
- [ ] **削除モード**実装:
  - 種類別チェックボックス (寸法/バルーン/タップ/図枠)
  - キャンバスクリックで個別選択
  - 「削除後プレビュー」
  - 削除実行 → サーバー側で該当エンティティ除外
- [ ] DXF書き出し（削除済みクリーン版）

**完了条件**: コベルコブームRBサンプルのバラ図を読込→不要要素削除→クリーンDXF書き出しまでが動く

---

## Phase 2 — 外径自動検出 + 加工代付加 (本命機能)

**目的**: 「材料取りCAD」の核となる機能

- [ ] **外形モード**実装:
  - トポロジ再構築ロジック (Shapely + 自作グラフ探索)
  - 信頼度算出
  - 自動検出成功 → シアン強調
  - 失敗 → 警告バナー + 手動選択モード
  - 線クリックチェーン選択 + 「閉じる」操作
- [ ] **加工代モード**実装:
  - 全周一律オフセット (pyclipper)
  - 辺ごと個別調整 (E1〜En)
  - 角の処理選択（円弧連結 / 鋭角延長）
  - プレビュー表示
  - 適用後の外周長・板取り寸法・材料効率%
- [ ] 加工代付きDXF書き出し
- [ ] **VPSに上げて触り心地確認** (SSHトンネル経由)

**完了条件**: ベテランの「手書き材料取り」と同等以上の結果が、誰でも生成できる

---

## Phase 3 — C面/開先 + PDF出力 + 図面整理

**目的**: 出力品質を上げて、現場で実用できる状態にする

- [ ] **C面モード**実装:
  - コーナーチップで角指定
  - C面サイズ・開先角度設定
  - DXF出力時の注記
- [ ] **PDF出力** (ReportLab):
  - 図枠なし / 紙印刷向け
  - 1ページ1部品
  - 材料取り専用枠（後日定義）の選択肢
- [ ] **図面整理機能**:
  - バルーン削除（既に削除モードでカバー）
  - 寸法線の自動再配置（必要寸法だけ残す）
  - 製作図枠 → 材料取り枠への置換

**完了条件**: 加工オペレーターが現場で使える出力が得られる

---

## Phase 4 — 社内展開 + 作図系ツール

**目的**: 多人数で使い、不足機能を追加する

- [ ] **Tailscale Funnel** でVPS公開（`https://cutflow.<tailnet>.ts.net`）、社内数人でテスト
- [ ] フィードバックに基づいて UI/UX 微調整
- [ ] **寸法モード** (Phase 4 で本実装):
  - 2点クリックで寸法線追加
  - ISO標準スタイル
- [ ] **線編集モード**:
  - 頂点ドラッグ
  - スナップ・グリッド吸着
- [ ] **穴追加モード**:
  - クリック配置
  - 整列パターン
- [ ] **注記モード**:
  - 加工指示テキスト
  - プリセット（面粗さ・溶接・一般）
- [ ] **ブリッジモード**:
  - 外径クリックで保持タブ配置

**完了条件**: 設計部・製造部の両者が日常業務で使えるツールに育つ

---

## Phase 5 — ネスティング + 運用機能

**目的**: 真に「材料取り最適化ツール」へ進化させる

- [ ] **ネスティング機能**:
  - 複数部品の自動配置（板取り最適化）
  - 歩留まり計算
  - ジョブキュー (重い処理を非同期化)
- [ ] **履歴管理**: セッション保存・呼び出し
- [ ] **テンプレート**: よく使う材質・板厚・加工代のプリセット
- [ ] **エクスポート拡張**: STEP / G-code 等への対応検討
- [ ] **監視・運用**: ログ集約、Discord通知連携（既存 lineworks-x-ops 経由）

**完了条件**: 材料取り工程全体が CutFlow•CAD で完結する

---

## 進捗判断のポイント

| 段階 | 「次に進んでよい」サイン |
|---|---|
| Phase 0 → 1 | 開発環境が手元で問題なく起動する |
| Phase 1 → 2 | 削除機能だけで「あって嬉しい」とフィードバックがある |
| Phase 2 → 3 | 外径オフセット結果がベテランの手書きと一致するか確認 |
| Phase 3 → 4 | 加工オペレーターから「これで現場OK」が出る |
| Phase 4 → 5 | 日常的に複数人が使い、機能不足が顕在化している |

---

## ノンゴール（やらないこと）

- 完全なAutoCAD互換性（材料取りに必要な機能だけ）
- 3D表示
- マルチテナント / 課金システム
- モバイル対応（社内PC専用）
- DXF以外の入力形式（当面）
